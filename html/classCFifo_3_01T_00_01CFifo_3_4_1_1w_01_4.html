void CFifo< T, CFifo<>::w >::commit ( ) [inline]

Commit a reserved buffer.

Note that this effectively only moves the write pointer to the next element in the FIFO queue.

It also means that if the current FIFO entry was not reserved and written, the committed data is undefined.

template<class T >
T* CFifo< T, CFifo<>::w >::reserve ( ) const [inline]

Reserves an element in the FIFO memory.

This will return a *write-only* pointer to the FIFO memory. Effectively, you can write data straight to the end point of the FIFO.

Mind that only char/short/int write operations can be used, so when T is a struct with bitfields and you would update a field, then the data will be read first!

Returns:
NULL if something went wrong or a pointer otherwise.
