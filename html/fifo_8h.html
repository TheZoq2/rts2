Detailed Description

C-HEAP kind of FIFO implementation using FIFO memories or shared heap.

In short, allocate two FIFO end points (or one shared), initialize the FIFO, create an RFifo and WFifo object--which is just a wrapper to access the FIFO in a nice way--at both end points and start pushing/popping data.

Allocates a FIFO end point in the local FIFO memory.

Since the FIFO itself is located at the read end point, there is no space allocated for the FIFO when the FIFO_ENDPOINT_WRITE flag is specified.

element_sizethe size of one element, as returned by sizeof()
flagstype of FIFO, a combination of the FIFO_* flags (but only the FIFO_ENDPOINT_READ and FIFO_ENDPOINT_WRITE are relevant)

a pointer to the allocated FifoEP, or NULL when there is not enough memory
