Detailed Description

Heap management.

This implementation is not thread safe. There is no check how much memory a process already has, so one process could consume all memory, preventing other processes of having some.

It would be nice to check the total allocated memory and prevent allocating too much memory. This improves composable behavior: processes do not influence each other. However, sharing the same heap leads to fragmentation, which leads to sub-optimal allocation, which can lead to less memory than expected in advance...

HEAP_NODE is quite large. Therefore, a lot of memory is consumed by administration. This must be improved.
