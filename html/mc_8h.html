struct mc_var ATTR_DATA_ALIGNED

Detailed Description

Implementation of a memory consistency model.

Specifically, either no caches are assumed or (kind of) entry consistency is implemented.

Different shared objects should not overlap in cache lines. To accomplish this, add ATTR_DATA_ALIGNED to all global variables, or smalloc() objects separately, or add ATTR_DATA_ALIGNED to the first element of a object (like a struct) in a typedef (regardless how the type is used). For pthread_mutex_t, pthread_cond_t and pthread_barrier_t, this has already been done are they are safe to use.
