bool button_pressed ( int  button)

Checks whether the button has been pressed.

It checks whether there is an event. When there is one, the current state of the button is checked. In case the button is currently pressed, it is assumed the event is a press event, which is then cleared and true is returned. When there was no event or the button isn't hold down currently, nothing is changed and false is returned.

Note that when a button is pressed and released before the call to button_pressed() is made, it is assumed that the last event was a release event, so false will be returned. In that case, the press event is lost. When it is required that all events are processed, make sure that button_pressed() and button_released() are called in a timely fashion. It might be a good practice to let software poll the registers using buttons_event() and call button_pressed() or button_released() when buttons_event() returns a non-zero value. Polling the buttons by repeatedly calling button_pressed() and buton_released() for all buttons to be monitored, is inefficient.

There is a race condition in the case that the button is pressed or released during the execution of this function. In that case, it is possible that old or new events are ignored. However, it is assumed that the buttons are just human input, which is not that fast to cause problems.

Returns:
true in the case of a new event and the button is currently pressed

unsigned int get_clock ( )

Returns the number of seconds and microseconds since the last reset.

The system contains a clock peripheral that has a 32-bit counter to count microseconds since the last reset. To prevent problems by reading the seconds and microseconds from different registers, both the seconds and microseconds are packed into the same 32-bit register. Therefore, the maximum number of seconds is limited by 2**(32-ceil(log2(1000000)))=4096.

Returns:
(nr_sec<<20 | nr_ms%%1000000)

void outbyte ( char  ch)

Writes one byte to the serial port.

This is a blocking call. It will poll the status register until space is available to write one byte. In case that two processes are writing simultaneously, race conditions occur, resulting in buffer overflows. The behavior is undefined when the UART fifo overflows. It is likely that the whole fifo is invalidated and previously written but not yet transmitted bytes are lost.
