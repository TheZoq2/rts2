00005  * \par \b Starburst - Helix
00006  *
00007  * The hardware supplies the following memory hierarchy:
00008  *
00009  * \li A local SRAM to every processor for single-cycle access to instructions and data.
00010  * \li A local memory that is write-only from the NoC and read/write from the local MicroBlaze.
00011  *     This memory is connected to the local PLB bus.
00012  * \li A global shared DDR SDRAM accessible for all processors, which can be cached by the MicroBlazes.
00013  *     This memory is connected to the NoC and is expensive in bandwidth, latency and power.
00014  *
00015  * The software uses the memory hierarchy as follows:
00016  *
00017  * \li All instructions are in DDR, which is cached; the .text section.
00018  * \li Half of the DDR is cached, which can be used for local data (the normal heap) and read-only data (.rodata section) and will be cached (without cache coherency).
00019  * \li The other half of the DDR is not cached, which can be used for global data (including the .bss section, .data and shared heap).
00020  *
00021  * For applications, this means that an explicit separation must be made for these type of variables:
00022  *
00023  * \li Function-local variables: allocate on the stack. Mind that the stack is small. The stack is malloc()-ed on the heap.
00024  * \li Application-local variables: you can use the stack, but this is limited in size, so use malloc() otherwise. The heap is cached and not cache coherent between cores.
00025  * \li Processor/kernel-local variables: the processor heap is shared among all processes, so this can be used for inter-process communication.
00026  * \li Global variables: use the shared heap (smalloc()) for that. Data will be put in uncached DDR, accessible by all cores, but very slow.
00027  *
00028  * Misc notes:
00029  *
00030  * \li When passing pointers between processes, they must be in uncached memory or the data cache must explicitly be flushed/invalidated.
00031  * \li Read-only data is punt in the cached .rodata section, read/write data in uncached .data. If a static variable is used as read-only, define it as 'const'.
00032  * \li There is no memory protection; writing to wrong addresses (including NULL) can result in really strange behavior (even after reset).
00033  * \li Stack overflows will not be detected sometimes, so keep you stack usage low and inspect it when necessary.
00034  * \li C++'s ::operator new() and ::operator new[]() are redirected to smalloc(), and ::operator delete() and ::operator delete[]() to sfree().
00035  * \li C++'s exceptions, virtual functions, etc. is kind of supported but not really tested.
00036  *
00037  * Compilation hints:
00038  *
00039  * \li The entry of your program is main(), as usual. This process is automatically started at core 0 when the system boots.
00040  * \li Compile everything statically.
00041  * \li Link against libhelix.a (which is done automatically when using starburst/bin/mb-g++).
00042  *
00043  * Running daemons at the time main() starts:
00044  *
00045  * \li DefaultTaskDaemon() (always process id 0)
00046  * \li MallocDaemon()
00047  * \li StartDumpStatsDaemon()
00048  * \li StartTimerDaemon() when you use a timer or sleep()
00049  * \li mainProc() which calls main() eventually
