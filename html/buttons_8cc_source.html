00057 /*!
00058  * \brief Checks whether the button has been pressed.
00059  *
00060  * It checks whether there is an event. When there is one, the current state of
00061  * the button is checked. In case the button is currently pressed, it is assumed
00062  * the event is a press event, which is then cleared and true is returned.
00063  * When there was no event or the button isn't hold down currently, nothing is changed
00064  * and false is returned.
00065  *
00066  * Note that when a button is pressed and released before the call to button_pressed()
00067  * is made, it is assumed that the last event was a release event, so false will be returned.
00068  * In that case, the press event is lost. When it is required that all events are processed,
00069  * make sure that button_pressed() and button_released() are called in a timely fashion.
00070  * It might be a good practice to let software poll the registers using buttons_event() and
00071  * call button_pressed() or button_released() when buttons_event() returns a non-zero value.
00072  * Polling the buttons by repeatedly calling button_pressed() and buton_released() for all
00073  * buttons to be monitored, is inefficient.
00074  *
00075  * There is a race condition in the case that the button is pressed or released during
00076  * the execution of this function. In that case, it is possible that old or new events are
00077  * ignored. However, it is assumed that the buttons are just human input, which is not
00078  * that fast to cause problems.
00079  *
00080  * \returns true in the case of a new event and the button is currently pressed
00081  */

00083     if(button_event(button)&&button_state(button)){
00084         button_clear_event(button);
00085         return true;
00086     }else
00087         return false;
