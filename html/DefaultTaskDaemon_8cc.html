Detailed Description

Daemon for communication and synchronization in the SoC.

The task daemon is the essential daemon of the OS; this is the first process running and is responsible for process management and synchronization for the running kernel and with other kernels in the system.

There is one very special command for the task daemon: the shutdown command. There is no call available to invoke this from a normal process--it should only be used for debugging. The call is (possibly) destructive for the whole system. No feedback is given when the core has been reset. It can take up to multiple replenishment intervals. All kernels wait for at least one second between terminating all processes and rebooting. Make sure that all cores that have been booted receive the shutdown command within that second to prevent race conditions.

Quick and dirty method to shutdown core p: 

 ((tdcmd_remote_t*)MAKE_NOC_FIFOMEM_PTR(0,p))->cmd[p].cmd=TDCMD_SHUTDOWN;  // (combined with next line)
 ((tdcmd_remote_t*)MAKE_NOC_FIFOMEM_PTR(0,p))->cmd[p].proc=p;              // (0x60000000 & (p<<20)) + (p*6+1)*4 = 0x1bff0000
 ((tdcmd_remote_t*)MAKE_NOC_FIFOMEM_PTR(0,p))->have_cmd=1;                 // (0x60000000 & (p<<20)) = 1

int dBootKernels ( int  num,
int *  procs 
)

Boots the kernel on other processors.

It does the following: other processors will get the boot signal (assuming the slaveloader is running) and we will wait for the boot notification. When you want to use the wrapper functions like CreateProcess(), all other cores must be booted using this function, in order to setup the channels correctly. Do not call this function twice.

Parameters:
numthe number of processors to boot, that is the length of the procs parameter
procsan array of core IDs of processors to boot
Returns:
0 on success, EXFULL when the fifo memory is full, otherwise an errno. In case of an error, it is undecidable which processors have been booted and which have not.
